Disk Health Checker

Setup guide for checking external hard drives on macOS

v0.2.0

What You Need

✓ A Mac (Intel or Apple Silicon)
✓ Terminal access
✓ Python 3.11+
✓ External drive plugged in
1

Install smartmontools

This gives you smartctl, the tool that reads SMART diagnostics from your drive.

Open Terminal and run:

brew install smartmontools
ℹ️
Don't have Homebrew? Install it first:
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

Verify it worked:

smartctl --version
# Should print: smartctl 7.4 ...
2

Download and install disk-health-checker

Clone the repository and install it in a virtual environment:

git clone https://github.com/quanb24/disk-health-checker.git
cd disk-health-checker
python3 -m venv .venv
source .venv/bin/activate
pip install -e .

Verify it worked:

disk-health-checker --version
# Should print: disk-health-checker 0.2.0
3

Plug in your external drive

Connect the USB drive and wait a few seconds for macOS to mount it.

4

See what drives are connected

disk-health-checker detect

This lists all connected disks with their names, sizes, and device paths (like /dev/disk4). Find your external drive in the list.

5

Run the health check

Interactive mode (easiest):

disk-health-checker smart

A numbered list of drives appears. Type the number for your external drive and press Enter.

Or specify the device directly:

disk-health-checker smart --device /dev/disk4

Replace /dev/disk4 with whatever detect showed for your drive.

6

Read your results

You'll get a clear verdict with a confidence rating, findings, and next steps:

PASS
Drive is healthy. No action needed.
WARNING
Early signs of wear. Keep an eye on it.
FAIL
Serious problems. Back up data immediately.
UNKNOWN
Not enough data. Common with some USB enclosures.
7

Get a beginner-friendly explanation

The doctor command explains every finding in plain language:

disk-health-checker doctor --device /dev/disk4
8

Save results as JSON

disk-health-checker smart --device /dev/disk4 --json > my-drive-report.json

Troubleshooting

Problem Fix
smartctl: command not found Run brew install smartmontools and restart Terminal
python3: command not found Install Python from python.org or run brew install python
Drive shows UNKNOWN verdict Some USB enclosures block SMART access — hardware limitation, not a bug
Permission denied Run with sudo disk-health-checker smart --device /dev/diskN
Drive not showing in detect Unplug and replug the drive, wait 5 seconds, try again

Quick Reference — After Setup

Every time you want to check a drive in the future, just run these three commands: 

cd disk-health-checker
source .venv/bin/activate
disk-health-checker smart
